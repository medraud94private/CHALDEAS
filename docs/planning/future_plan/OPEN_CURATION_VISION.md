# Open Curation Vision

> **상태**: V2 구상
> **작성일**: 2026-01-07

---

## 핵심 아이디어

큐레이션 레이어를 오픈소스로 공개하여 커뮤니티 기반 역사 큐레이션 실현.

**영감**: 나무위키, 위키피디아의 하이퍼링크 기반 지식 연결 방식

---

## 컨셉

```
CHALDEAS 코어 (비공개)          Open Curation (공개)
┌─────────────────────┐      ┌─────────────────────┐
│ 데이터 수집/처리     │      │ 큐레이터 편집 인터페이스│
│ NER 파이프라인       │  →   │ 하이퍼링크 연결       │
│ Historical Chain 생성│      │ 원문 자료 첨부        │
│ World State (불변)   │      │ 주석/해석 작성        │
└─────────────────────┘      └─────────────────────┘
```

---

## 큐레이터가 할 수 있는 것

1. **글 작성**: 자신만의 역사 해석/스토리 작성
2. **링크 연결**: 다른 글, Historical Chain, 원문 자료 하이퍼링크
3. **원문 첨부**: 1차 사료, 논문 등 직접 연결
4. **Chain 보완**: AI가 생성한 Chain에 맥락 추가

---

## 아키텍처 원칙

- **Immutable Core**: 원본 데이터/Chain은 수정 불가
- **Curation as Overlay**: 큐레이션은 원본 위에 덧씌우는 레이어
- **출처 투명성**: 모든 연결은 추적 가능

---

## 해석 충돌 정책

같은 사건/인물에 대해 다른 해석이 등록될 경우:

| 방식 | 설명 |
|------|------|
| 병렬 표시 | 모든 해석을 동등하게 나열, 사용자가 선택 |
| 출처 명시 | 각 해석의 근거 자료 필수 첨부 |
| 전문가 태그 | 검증된 전문가 해석에 별도 표시 |

```
예: "임진왜란 발발 원인"
├── 해석 A: 경제적 요인 (출처: 논문 X)
├── 해석 B: 정치적 요인 (출처: 논문 Y)
└── 해석 C: 복합 요인 (출처: 저서 Z)
→ 사용자가 모든 관점 열람 가능
```

**원칙**: 단일 정답 강요 X, 다양한 해석 공존

## 삭제/수정 권한 정책

"Immutable Core" 원칙과 사용자 권리 충돌 해결:

| 상황 | 처리 방식 |
|------|----------|
| 큐레이션 수정 | 버전 히스토리 유지, 이전 버전 열람 가능 |
| 큐레이션 삭제 요청 | 비공개 전환 (데이터는 보존) |
| 개인정보 삭제 요청 | 메타데이터 익명화, 콘텐츠는 유지 |
| 저작권 침해 신고 | 즉시 비공개 → 검토 후 처리 |

```
삭제 ≠ 완전 제거
삭제 = 비공개 전환 + 익명화
```

**법적 고려**: GDPR 등 규정 준수 필요
